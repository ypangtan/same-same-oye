Introduction

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

http://localhost/yobe-backoffice

Authenticating requests

This API is not authenticated.

Order Operation API

1. Update order status (Scan Order)

Example request:
curl --request POST \
    "http://localhost/yobe-backoffice/api/v1/order-operation/update-order-status" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --header "X-Vending-Machine-Key: string secret key of the machine to request verification. Example: 123ifa9sdb1j23sf" \
    --data "{
    \"reference\": \"1knkbasbmc\"
}"

const url = new URL(
    "http://localhost/yobe-backoffice/api/v1/order-operation/update-order-status"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
    "X-Vending-Machine-Key": "string secret key of the machine to request verification. Example: 123ifa9sdb1j23sf",
};

let body = {
    "reference": "1knkbasbmc"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request   

POST api/v1/order-operation/update-sales-data

Body Parameters

sales_date  string  

The date of sales in YYYY-MM-DD format.

sales_type  integer optional  

The type of sales (e.g., 1 for daily, 2 for weekly, 3 for monthly). Default: 1

sales_metas  string[] optional  

Additional sales metadata (e.g., products sold).

order_references  string[] optional  

A list of related order references.
